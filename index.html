Father Fouras question engine

This small web-app was created to aid the support-team at Visit with the everyday tasks that previosly involved SQL-queries in the database.
The application was set up as an MVC ASP.NET-app with the sole purpose of visualizing SQL-queries. The queries performed included org:[id], product:[id] etc.
With the resulting pages, the support staff could check distribution channels per organization, what settings that were set behind the scenes, how many products that were bookable and present online (for this, an ajax-call was utilized which returned live-status).

The application has been used a lot, and started as a Hackathon-project. I've also learnt a lot about dictionaries, loops and linq-queries in this process.


This is the organization-result. Showing stats, distribution channels and suppliers/subscribers of the organization. Everything clickable to continue searching.



This is the channel-result, paginated with every active product present in the channel. Extra points if bookable and shown to the right.



This is the per-product result. It shows various stats including categories, children/parents, subscription, but also every distribution channel and if possible, live status online.
